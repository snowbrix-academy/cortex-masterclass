Anomaly Detection

Two Methods:

  • Z-Score Method: Flag days with spend >2σ above 30-day rolling average
  • Percentage Spike: Compare today vs same-day-last-week, flag if >50% increase

SP_DETECT_ANOMALIES runs daily, identifies unusual spend patterns automatically

Month-End Forecasting

  • Calculate burn rate: daily average spend × days remaining
  • Linear projection to month-end
  • Compare forecast vs budget
  • Alert if projected to exceed budget by >10%

Example: On Feb 15th with $30K spent, burn rate = $2K/day, forecast = $30K + ($2K × 13 days) = $56K projected month-end
