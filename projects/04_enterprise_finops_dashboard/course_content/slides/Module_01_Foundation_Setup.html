Two-Database Architecture

FINOPS_CONTROL_DB

Single Source of Truth

• Raw cost facts from ACCOUNT_USAGE
• Stored procedures & business logic
• Chargeback attribution dimensions
• Budget definitions & alerts
• Configuration & metadata

FINOPS_ANALYTICS_DB

Performance Layer

• Pre-aggregated metrics (daily/weekly/monthly)
• Optimized for BI tool consumption
• Prevents expensive ad-hoc queries
• Different retention policies
• Independent scaling

Why separate? BI tools query ANALYTICS_DB (fast, pre-aggregated) while procedures write to CONTROL_DB (isolated, no contention).

Warehouse Sizing Principles

Best Practices

  • Start small — XSMALL/SMALL first
  • Aggressive auto-suspend for batch workloads (60s)
  • Multi-cluster for concurrency, not speed
  • Separate concerns — ETL vs reporting

Common Mistakes

  • Starting with LARGE — 8x waste
  • Long auto-suspend (10+ minutes)
  • Single warehouse for everything
  • Multi-cluster for batch ETL

Sizing Rule: Monitor query queue time. If consistently <1 second, warehouse is right-sized or oversized. If >5 seconds, consider upsize.

Four-Role RBAC Model

ACCOUNTADMIN (Snowflake built-in) └── SYSADMIN (Snowflake built-in) └── FINOPS_ADMIN_ROLE (full framework access) ├── FINOPS_ANALYST_ROLE (read-only all data) ├── FINOPS_TEAM_LEAD_ROLE (read-only team data) └── FINOPS_EXECUTIVE_ROLE (read-only summary)

Least-Privilege Principle: Assign users to the lowest-privilege role that meets their needs. Review role membership quarterly.

Troubleshooting Common Issues

Error: "Insufficient privileges"

Cause: Not using ACCOUNTADMIN role for initial setup

Solution: USE ROLE ACCOUNTADMIN; before running scripts

Error: "Object already exists"

Cause: Re-running scripts in same account

Solution: Safe to ignore — IF NOT EXISTS clauses prevent errors

Warehouses not showing MIN_CLUSTER_COUNT

Cause: Only multi-cluster warehouses have this property

Solution: Normal — only FINOPS_WH_REPORTING is multi-cluster

Foundation Best Practices

Database Design

  • Separate control from analytics (performance)
  • Logical schema boundaries (security)
  • Future grants for automation

Warehouse Design

  • Start small, scale if needed
  • Aggressive auto-suspend for batch
  • Multi-cluster for concurrency only

Security Design

  • Least-privilege role assignment
  • Role hierarchy for inheritance
  • Quarterly access reviews

Cost Control

  • Resource monitor before resuming
  • Conservative initial quota (50 credits)
  • Monitor and adjust monthly
