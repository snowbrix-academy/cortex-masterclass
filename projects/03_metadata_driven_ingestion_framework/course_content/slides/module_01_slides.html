Module 01 — Databases

Multi-Layer Database Architecture

MDF_CONTROL_DB (Framework Brain) +-- CONFIG Config tables, registries +-- AUDIT Run history, error logs +-- PROCEDURES All stored procedures +-- MONITORING Dashboard views MDF_RAW_DB (Landing Zone - Immutable) +-- <CLIENT>_<SOURCE> Dynamic schemas per source MDF_STAGING_DB (Cleaned & Validated) +-- <CLIENT>_<SOURCE> Mirrors raw with quality checks MDF_CURATED_DB (Business-Ready) +-- <CLIENT>_<DOMAIN> Transformed, aggregated

Rule: Never transform data in RAW. It's your safety net.

Module 01 — Databases

Why Separate Databases?

Security Isolation

Grant analyst access to CURATED without exposing RAW. Each layer has different RBAC requirements.

Data Lineage

Clear path: RAW → STAGING → CURATED. You can always trace where data came from.

Cost Attribution

Snowflake bills by database. Separate databases = separate cost buckets for finance.

Lifecycle Management

Different Time Travel settings per layer. RAW: 1 day. STAGING: 7 days. CURATED: 90 days.

Key principle: Never size a warehouse based on your biggest table. Size it based on your query complexity. COPY INTO is I/O bound — SMALL is usually enough.

Resource Monitors Are Not Optional

CREATE RESOURCE MONITOR MDF_INGESTION_MONITOR WITH CREDIT_QUOTA = 100 -- 100 credits per month FREQUENCY = MONTHLY TRIGGERS ON 75 PERCENT DO NOTIFY -- Alert at 75% ON 90 PERCENT DO NOTIFY -- Alert at 90% ON 100 PERCENT DO SUSPEND -- Stop at 100% ON 110 PERCENT DO SUSPEND_IMMEDIATE;

Without resource monitors: A single misconfigured task can burn through your entire monthly budget overnight. We've seen it happen. Multiple times.

Module 01 — RBAC

Future Grants — Set Once, Works Forever

-- Without future grants: every new table needs manual grants GRANT SELECT ON TABLE new_table TO ROLE MDF_READER; -- For EVERY new table! -- With future grants: new objects auto-inherit permissions GRANT SELECT ON FUTURE TABLES IN DATABASE MDF_CURATED_DB TO ROLE MDF_READER; -- ^ Set this ONCE. Every future table gets SELECT automatically.
  • Functional roles (MDF_LOADER) not user roles (JOHN_ROLE)
  • Future grants prevent permission drift over time
  • Least privilege — LOADER can only write to RAW
